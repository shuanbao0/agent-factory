# 项目标准 — 全局项目执行规范

## LIFECYCLE

### requirements
**入口条件：** 项目已创建，BRIEF.md 已生成
**出口条件：** 需求文档完成，所有关键需求已确认，验收标准已定义
**交付物：**
- 需求文档（requirements.md 或等价物）
- 验收标准清单

### design
**入口条件：** 需求文档已完成并确认
**出口条件：** 设计方案已确定，技术选型完成，接口/结构已定义
**交付物：**
- 设计文档（architecture.md / design.md）
- 关键接口或数据结构定义

### development
**入口条件：** 设计方案已确认
**出口条件：** 核心功能已实现，代码可运行，基本自测通过
**交付物：**
- 功能代码
- 基本测试或验证结果

### testing
**入口条件：** 核心功能已实现
**出口条件：** 所有测试通过，已知问题已记录，质量达到交付标准
**交付物：**
- 测试报告
- 已知问题清单（如有）

### delivery
**入口条件：** 测试通过，质量达标
**出口条件：** 产出已交付到指定位置，文档已更新，项目状态已标记完成
**交付物：**
- 最终交付物（代码/文档/报告等）
- 项目总结

## BOUNDARIES

### DO
- 所有产出写入项目目录 `projects/{dept}/{slug}/`
- 每阶段完成后更新 `.project-meta.json` 的 `currentPhase`
- 遵循项目目录约定（各子目录用途明确）
- 在项目目录中留下协作笔记供其他 Agent 参考
- 重要决策记录在项目文档中

### DON'T
- 不跳过需求阶段直接编码
- 不修改项目目录外的文件（除 workspaces/ 中的个人产出）
- 不在未确认设计方案的情况下开始大规模开发
- 不删除其他 Agent 的产出文件
- 不跳过测试阶段直接交付
