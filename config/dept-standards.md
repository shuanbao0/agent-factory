# 部门执行标准 — 全局部门行动原则

## GENERAL

- **先完成再开始** — 已有进行中任务的 agent 不派新活，专注完成现有工作
- **待办优先于新建** — 优先分配已存在的 pending 任务，减少任务膨胀
- **无任务的空闲 agent 必须有事做** — 发现无任务的空闲 agent 不分配是失职
- 卡住超过 2 轮的任务要换方式推进或换人
- 重要进展和阻塞立即上报

## TYPES

### novel
- **创意优先** — 鼓励创新表达，评审侧重故事性和文学性而非格式
- **连贯性检查** — 角色设定、世界观、情节线必须跨章节一致
- **风格统一** — 同一作品内文风、叙事视角保持一致
- 创作类任务允许更长的执行周期，不要因短期无产出就判定失败
- worldbuilding 和 character 任务必须在 writing 之前完成

### dev
- **代码质量** — 评审必须检查可运行性、测试覆盖、安全性
- **架构一致** — 新代码必须与现有架构风格一致，避免引入不兼容的模式
- **文档同步** — API 变更必须同步更新文档
- 技术债务及时标记和追踪，不在功能开发中顺手大规模重构
- coding 任务必须有基本测试才能进入 review

### research
- **证据驱动** — 结论必须有数据或文献支撑，标注来源
- **全面性** — 调研覆盖主流方案和替代方案，不能只看一个方向
- **可操作性** — 研究产出必须包含明确的建议和下一步行动

### finance
- **合规优先** — 所有产出必须符合合规要求，合规审查不可跳过
- **数据准确** — 数字和计算必须经过交叉验证
- **审计可追溯** — 决策依据和数据来源必须记录，便于事后审计

### quant
- **回测覆盖** — 策略必须在多种市场环境下回测验证
- **风险评估** — 策略修改必须附带风险分析和最大回撤评估
- **数据质量** — 使用的市场数据必须验证完整性和准确性

### brand
- **品牌一致** — 所有产出必须符合品牌调性和视觉规范
- **受众导向** — 内容创作以目标受众需求为核心
- **创意与合规并重** — 创意表达不能违反法规和品牌底线

### ops
- **流程完整** — 运营方案必须覆盖全流程，不遗漏关键环节
- **可执行性** — 方案必须有明确的责任人、时间节点、验收标准
- **异常预案** — 必须考虑异常情况的处理方案

### tutorial
- **准确性** — 技术内容必须准确无误，示例代码必须可运行
- **循序渐进** — 内容编排从简到难，不跳跃关键步骤
- **读者视角** — 从读者角度验证每个步骤的可复现性

### anime
- **制作规范** — 遵循动画制作流水线标准（分镜→原画→上色→后期）
- **风格统一** — 同一作品内角色设计、场景风格保持一致
- **进度协调** — 各环节产出必须按时交付，不阻塞下游

### sales
- **客户导向** — 所有方案以客户需求和痛点为核心
- **数据支撑** — 销售策略和预测必须有数据依据
- **合规底线** — 不夸大产品能力，不做虚假承诺

### service
- **响应及时** — 客户问题必须在规定时间内响应
- **解决导向** — 以解决问题为目标，不推诿不拖延
- **记录完整** — 所有客户交互必须有完整记录

### legal
- **严谨准确** — 法律文件和意见必须准确无误
- **风险预警** — 主动识别法律风险并提出防范建议
- **时效敏感** — 关注法规变更和截止日期
