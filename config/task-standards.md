# 任务标准 — 全局任务执行规范

## GENERAL

### 完成定义
- 产出文件存在且内容完整
- 通过自检评分 ≥ 策略阈值
- 产出可被下一环节直接使用

### 质量检查清单
1. 是否完成了任务要求的所有内容？
2. 是否有明显的错误或遗漏？
3. 格式和表述是否规范？
4. 是否可以交付给下一环节？

### DO
- 完成后通过 API 更新任务状态
- 所有产出写入 workspaces/ 或项目目录
- 遇到阻塞立即通过 peer-send 通知主管
- 产出中注明关键假设和决策依据

### DON'T
- 不提交未自检的产出
- 不修改任务范围外的文件
- 不在未理解需求的情况下动手

## TYPES

### writing
**完成定义：** 文稿完整，无未完成章节，字数达到要求
**质量检查清单：**
1. 情节是否连贯，无逻辑断裂？
2. 文笔质量是否达标（语言流畅、表达准确）？
3. 字数是否达到要求？
4. 是否遵循了世界观和角色设定？
5. 是否与前文保持一致（人物、地名、时间线）？
**DO：** 参考角色设定和世界观文档，遵循大纲
**DON'T：** 不偏离大纲，不引入矛盾设定，不重复已写内容

### coding
**完成定义：** 代码可运行，有基本测试，无明显安全漏洞
**质量检查清单：**
1. 代码是否能编译/运行无错误？
2. 核心逻辑是否有单元测试？
3. 是否有安全漏洞（注入、XSS 等）？
4. 代码风格是否与项目一致？
5. 是否有适当的错误处理？
**DO：** 遵循项目编码规范，编写测试，处理边界情况
**DON'T：** 不引入未经审批的第三方依赖，不硬编码敏感信息，不跳过测试

### research
**完成定义：** 研究报告完整，数据来源可靠，结论有据可依
**质量检查清单：**
1. 数据来源是否可靠且已标注？
2. 分析是否全面覆盖了研究问题？
3. 结论是否有数据支撑？
4. 是否提供了可操作的建议？
**DO：** 标注数据来源，区分事实与推断，提供多角度分析
**DON'T：** 不编造数据，不遗漏关键信息源，不做无依据的推断

### analysis
**完成定义：** 分析报告结构清晰，数据准确，洞察有价值
**质量检查清单：**
1. 数据是否准确无误？
2. 分析框架是否合理？
3. 洞察是否有实际价值？
4. 可视化/表格是否清晰？
**DO：** 使用结构化分析框架，数据交叉验证，提供可视化
**DON'T：** 不忽略异常数据，不过度解读相关性为因果性

### editing
**完成定义：** 编辑修改完成，修改痕迹清晰，质量有提升
**质量检查清单：**
1. 修改是否改善了原文质量？
2. 是否保持了原文风格和意图？
3. 修改是否引入了新的错误？
4. 格式和排版是否统一？
**DO：** 保留修改记录，说明修改理由，保持原文风格
**DON'T：** 不改变原文核心意图，不引入个人风格偏好

### worldbuilding
**完成定义：** 世界观设定完整，内部逻辑自洽，可供创作参考
**质量检查清单：**
1. 设定是否内部逻辑自洽？
2. 是否与已有设定兼容？
3. 是否提供了足够的细节供创作使用？
4. 是否考虑了设定对情节的影响？
**DO：** 与已有设定交叉验证，考虑设定的连锁影响
**DON'T：** 不与已确认的设定矛盾，不创建过于复杂无法维护的设定

### character
**完成定义：** 角色设定完整，性格鲜明，行为逻辑可信
**质量检查清单：**
1. 角色性格是否鲜明且一致？
2. 角色动机是否合理？
3. 角色关系是否清晰？
4. 角色是否有成长空间？
**DO：** 确保角色与世界观匹配，考虑角色间的化学反应
**DON'T：** 不创建扁平化角色，不忽略角色的内在矛盾

### plotting
**完成定义：** 情节大纲完整，节奏合理，冲突有张力
**质量检查清单：**
1. 情节线是否完整（起承转合）？
2. 节奏是否合理（松紧有度）？
3. 冲突是否有足够张力？
4. 伏笔和回收是否对应？
**DO：** 考虑多条情节线的交织，设计合理的伏笔
**DON'T：** 不留未解决的情节线，不设置无法回收的伏笔
