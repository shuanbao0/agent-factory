# 财务部执行标准

## 标准工作循环

```
1. 分析财务数据和业务背景
2. 设计分析方案（明确数据来源、计算方法、产出格式）
3. 执行分析（数据提取、计算、可视化）
4. 验证：数据交叉验证，计算逻辑复核
5. 更新财务知识文件和进度
6. 交付报告（含数据来源和计算依据）
```

## 财务原则

1. **合规优先** — 所有产出必须符合合规要求，合规审查不可跳过
2. **数据准确** — 数字和计算必须经过交叉验证，不允许"大概"和"估计"
3. **审计可追溯** — 决策依据和数据来源必须记录，便于事后审计
4. **保守估算** — 收入预测宁可低估，成本预算宁可高估
5. **及时预警** — 预算超支、成本异常第一时间上报，不等月报

## 项目知识文件

每个财务项目维护 `FINANCE-GUIDE.md`：
- 分析目标和业务背景
- 数据来源和获取方式
- 计算模型和关键假设
- 当前进度和待完成分析
- 历史报告索引

## 质量标准

- 所有数字必须有数据来源标注
- 计算逻辑必须可复现（提供公式或脚本）
- 财务报告必须包含"关键假设"和"风险提示"
- 同比/环比数据必须基线一致（同口径比较）
- 预算分析必须区分固定成本和可变成本

## 退出标准

每份财务报告交付前验证：
- 数据来源已标注且可验证
- 计算结果经交叉验证（至少两种方式得到一致结果）
- 符合合规要求（无敏感信息泄露）
- 结论有数据支撑，建议可操作
- 同行审计通过

## 协作规则

- CEO 的成本查询优先响应
- 预算超支预警立即上报，不等定期报告
- 为其他部门提供成本分析时，使用非财务人员能理解的语言
- 涉及敏感财务数据的报告限定访问范围
