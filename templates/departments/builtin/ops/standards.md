# 运营部执行标准

## 标准工作循环

```
1. 分析运营数据和市场趋势
2. 设计运营方案（渠道、内容、投放策略）
3. 执行分发和推广
4. 监控数据（流量、转化、ROI）
5. 优化调整（A/B 测试、策略迭代）
6. 交付运营报告和下周期计划
```

## 运营原则

1. **数据驱动** — 所有运营决策基于数据，不凭直觉和经验
2. **ROI 导向** — 每项运营活动必须有可衡量的 ROI 指标
3. **平台适配** — 不同平台（抖音、快手、B站、小红书）有不同的内容策略和分发规则
4. **快速迭代** — 小规模测试→数据验证→规模化，不大规模投放未验证方案
5. **用户增长优先** — 当前阶段以获客和留存为核心目标

## 项目知识文件

每个运营周期维护 `OPS-GUIDE.md`：
- 目标用户画像和获客渠道
- 各平台账号和内容策略
- 当前运营数据基线（DAU、转化率、获客成本）
- 内容日历和排期
- A/B 测试记录和结论

## 质量标准

- 内容发布前检查平台规则合规性
- 数据分析必须区分相关性和因果性
- A/B 测试必须有足够样本量才能下结论
- 运营报告必须包含同比/环比数据
- 投放预算和实际消耗必须对账

## 退出标准

每个运营周期完成时验证：
- 核心数据指标已记录（流量、转化、成本）
- 有效策略已沉淀为可复用 SOP
- 无效策略已标记并说明原因
- 下周期计划已制定
- 运营知识文件已更新

## 协作规则

- 内容需求提给品牌部或创作部门，附带平台要求和数据标准
- 产品反馈（用户评论、差评分析）定期同步给 PM
- 付费投放预算需 CEO 审批
- 跨平台数据整合后统一分析，不分平台各自为政
