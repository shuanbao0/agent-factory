# 量化交易部执行标准

## 标准工作循环

```
1. 研究市场数据和交易信号
2. 设计策略模型（明确假设、参数、风险边界）
3. 回测验证（多周期、多市场环境）
4. 风险评估（最大回撤、夏普比率、极端场景）
5. 模拟盘验证 → 实盘部署
6. 持续监控和策略优化
```

## 量化原则

1. **回测覆盖** — 策略必须在多种市场环境下回测验证（牛市、熊市、震荡市）
2. **风险评估** — 策略修改必须附带风险分析和最大回撤评估
3. **数据质量** — 使用的市场数据必须验证完整性和准确性，警惕幸存者偏差
4. **严格风控** — 仓位管理、止损机制、最大持仓比例必须在策略中内置
5. **不过拟合** — 回测结果过于完美时必须怀疑过拟合，用样本外数据验证

## 项目知识文件

每个策略维护 `STRATEGY-GUIDE.md`：
- 策略假设和市场观点
- 模型参数和调参历史
- 回测结果（多周期对比）
- 风险指标（夏普比率、最大回撤、胜率、盈亏比）
- 实盘表现跟踪

## 质量标准

- 回测必须覆盖至少 3 种市场环境（趋势、震荡、极端）
- 回测数据不少于 2 年历史（加密市场至少 1 年）
- 策略必须有明确的进场/出场/止损规则，不留模糊地带
- 风险指标必须量化：最大回撤、夏普比率、胜率、平均盈亏比
- 新策略上线前必须有至少 2 周模拟盘验证

## 退出标准

每个策略阶段完成时验证：
- 回测报告完整（含多市场环境结果）
- 风险评估通过（最大回撤在可接受范围内）
- 代码经过同行审核（计算逻辑、边界处理）
- 模拟盘数据与回测结果基本一致
- 策略文档已更新

## 协作规则

- 策略上线和参数调整需 Chief 审批
- 市场异常（黑天鹅）时触发紧急风控流程，不等循环
- Data Engineer 负责数据质量，Researcher 负责策略逻辑，两者职责不混
- 实盘策略修改必须走完整的回测→模拟→上线流程，不热更新
- 风险指标超过阈值自动告警，不依赖人工巡检
