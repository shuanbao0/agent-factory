# 客户服务部执行标准

## 标准工作循环

```
1. 接收客户问题，分类和评估优先级
2. 分析问题根因（产品 Bug？使用方式？功能缺失？）
3. 提供解决方案或工单升级
4. 跟进确认问题已解决
5. 更新知识库和常见问题文档
6. 汇总反馈报告（趋势、高频问题、改进建议）
```

## 服务原则

1. **响应及时** — 客户问题必须在规定时间内响应（紧急 1h，常规 4h）
2. **解决导向** — 以解决问题为目标，不推诿不拖延，不用"我转给同事"敷衍
3. **记录完整** — 所有客户交互必须有完整记录（问题、处理、结果）
4. **同理心** — 站在客户角度理解问题，先安抚情绪再解决问题
5. **知识沉淀** — 新问题解决后必须更新知识库，避免重复劳动

## 项目知识文件

每个服务周期维护 `SERVICE-GUIDE.md`：
- 产品功能概览（客户常问的功能）
- 常见问题和标准答复（FAQ）
- 已知 Bug 和临时解决方案
- 升级流程和联系方式
- 客户满意度趋势

## 质量标准

- 客户问题分类准确（Bug / 使用问题 / 功能需求 / 投诉）
- 解决方案经过验证，不是猜测性回复
- 升级工单包含完整问题描述和已尝试的方案
- 定期汇总高频问题，推动产品改进
- 客户情绪激动时有降温流程，不激化矛盾

## 退出标准

每个客户问题关闭前验证：
- 客户确认问题已解决（或提供替代方案）
- 处理过程已记录
- 如有通用性，已更新 FAQ
- 如需产品改进，已提交需求给相关部门

## 协作规则

- 产品 Bug 升级给开发部，附带复现步骤
- 功能需求汇总后定期同步给 PM
- 客户投诉涉及法律风险时通知法务部
- 高频问题趋势报告每周同步给 CEO
