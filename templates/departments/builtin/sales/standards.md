# 销售部执行标准

## 标准工作循环

```
1. 分析目标客户和市场机会
2. 设计销售策略（渠道、话术、定价方案）
3. 执行客户触达和跟进
4. 验证：跟踪转化数据，评估策略效果
5. 更新客户知识库和销售进度
6. 交付成交报告或阶段总结
```

## 销售原则

1. **客户导向** — 所有方案以客户需求和痛点为核心，不强推不匹配的产品
2. **数据驱动** — 销售策略和预测必须有数据依据，不凭直觉
3. **合规底线** — 不夸大产品能力，不做虚假承诺，不使用误导性信息
4. **流程可追踪** — 每个客户从线索到成交的全过程有记录
5. **反馈闭环** — 客户反馈和市场信号及时同步给产品和研发团队

## 项目知识文件

每个销售项目/周期维护 `SALES-GUIDE.md`：
- 目标客户画像和市场定位
- 产品卖点和竞品对比
- 销售漏斗当前状态
- 常见客户异议和应对话术
- 成交案例和失败复盘

## 质量标准

- 客户需求分析必须有实际调研支撑（非臆测）
- 报价方案必须有成本核算和利润分析
- 竞品分析必须客观准确，不贬低竞品
- 销售预测必须注明假设条件和置信度
- 每次客户交互有记录（时间、内容、下一步）

## 退出标准

每个销售阶段完成时验证：
- 客户信息完整且准确
- 下一步行动明确（时间、负责人、内容）
- 阶段转化数据已记录
- 客户知识库已更新

## 协作规则

- 售前技术支持由 Presales 提供，销售不承诺未确认的技术方案
- 市场反馈和客户需求定期同步给品牌部和产品团队
- 大客户/战略客户需要 CEO 确认后才能给出特殊报价
- 成交案例整理后共享给全团队，失败案例也要复盘
