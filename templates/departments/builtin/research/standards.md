# 研究院执行标准

## 标准工作循环

```
1. 分析研究领域现状和已有成果
2. 设计研究方案（明确假设、方法、预期产出）
3. 执行研究（文献调研 / 实验 / 数据分析）
4. 验证结论（交叉验证、同行评审）
5. 更新研究笔记和进度
6. 交付研究报告
```

## 研究原则

1. **证据驱动** — 所有结论必须有数据或文献支撑，标注来源和可信度
2. **全面性** — 调研覆盖主流方案和替代方案，不只看一个方向
3. **可操作性** — 研究产出必须包含明确的建议和下一步行动，避免纯学术研究
4. **商业化导向** — 聚焦能降低成本或提升质量的实用技术，优先支持其他部门
5. **可复现** — 实验方法和数据处理过程必须可复现，第三方能验证

## 项目知识文件

每个研究项目维护 `RESEARCH-GUIDE.md` 作为知识中枢：
- 研究目标和背景
- 已有发现和关键文献
- 当前进度和待验证假设
- 技术栈和工具链
- 协作依赖（哪些部门在等结论）

## 质量标准

- 数据来源必须标注（论文引用、实验数据、公开数据集）
- 对比分析必须有基线（baseline）和控制变量
- 实验结果必须包含置信度或误差范围
- 研究报告必须有"局限性"和"未来方向"章节
- 代码类实验必须可复现（提供环境配置和运行步骤）

## 退出标准

每个研究阶段完成时验证：
- 文献综述覆盖主流方案（≥5 个相关工作）
- 实验数据完整且可复现
- 结论有数据支撑，非主观判断
- 同行评审通过（peer-review 环节）
- 研究笔记已更新

## 协作规则

- 其他部门的技术支持请求优先于内部研究课题
- 研究成果必须转化为可执行的技术方案，不能止步于报告
- 跨部门协作时提供简化版结论（非完整论文），方便执行团队理解
