# 教程创作部执行标准

## 标准工作循环

```
1. 分析目标读者和技术背景
2. 设计教程大纲（从简到难，明确前置知识）
3. 撰写内容 + 编写可运行示例代码
4. 验证：每个步骤可复现，示例代码可运行
5. 更新教程知识文件和进度
6. 交付发布
```

## 创作原则

1. **读者视角** — 从读者角度验证每个步骤的可复现性，不假设未说明的前置知识
2. **循序渐进** — 内容编排从简到难，不跳跃关键步骤
3. **准确性** — 技术内容必须准确无误，示例代码必须可运行
4. **实用性** — 优先讲解实际应用场景，减少抽象理论
5. **持续更新** — 技术栈版本变更时及时更新教程内容

## 项目知识文件

每个教程系列维护 `TUTORIAL-GUIDE.md`：
- 目标读者画像和前置知识要求
- 教程系列大纲和章节规划
- 当前进度和待完成章节
- 技术栈版本和环境要求
- 代码示例仓库位置

## 内容质量标准

- 每篇教程有明确的学习目标（"读完本章你将学会..."）
- 代码示例必须**完整可运行**，不能只展示片段
- 关键步骤必须有截图或输出示例
- 覆盖常见错误和排查方法（"如果你看到这个错误..."）
- 前置条件和环境要求在教程开头说明
- 字数 ≥1500（质量门要求），重复率 ≤25%

## 退出标准

每篇教程发布前验证：
- 全部代码示例在干净环境中可运行
- 步骤按顺序执行无遗漏
- 同行评审通过（非作者按步骤复现一遍）
- 无技术错误和过时信息
- 教程知识文件已更新

## 协作规则

- Researcher 提供技术调研后，Writer 才能开始撰写
- Reviewer 必须实际运行代码，不能只看文字
- Code Instructor 负责示例代码质量，Writer 负责文字表述
- Visual Editor 的图表和截图必须与最新代码版本一致
