# 网文创作部执行标准

## 标准工作循环

```
1. 分析作品定位、目标平台和读者群体
2. 设计创作方案（世界观 → 角色设定 → 大纲 → 章节计划）
3. 按大纲顺序执行写作
4. 验证：连贯性检查、文笔质量、字数达标
5. 更新作品知识文件和创作进度
6. 交付章节（发布 / 提交审核）
```

## 创作原则

1. **先设定后写作** — worldbuilding 和 character 任务必须在 writing 之前完成
2. **线性推进** — 章节按大纲顺序执行，不跳章，不回头大改已发布章节
3. **连贯性至上** — 角色设定、世界观、情节线必须跨章节一致
4. **文学性优先** — 评审侧重故事性和文学性，而非格式规范
5. **持续产出** — 保持稳定更新节奏，创作类任务允许更长周期但必须有阶段产出

## 项目知识文件

每部作品维护 `NOVEL-GUIDE.md` 作为创作中枢：
- 作品定位（题材、平台、目标读者、字数规划）
- 世界观设定（地理、势力、规则体系）
- 角色设定表（主角/配角的性格、背景、关系网）
- 大纲和章节计划（含各章核心事件和字数预期）
- 创作进度（已完成/进行中/待写章节）
- 文风指南（叙事视角、语言风格、禁忌表述）

**每章完成后更新进度**，每卷完成后更新角色关系和世界观变化。

## 创作流程

- 章节写作必须按大纲顺序执行，不跳章
- worldbuilding 和 character 任务必须在 writing 任务之前完成
- 每章完成后需与大纲核对一致性
- 同一作品内文风、叙事视角保持统一

## 质量标准

- 避免流水账式叙述，注重场景描写和对话质量
- 角色言行必须符合其性格设定（不 OOC）
- 伏笔和悬念有埋设有回收，不烂尾
- 最终产出字数不低于大纲预设的 80%
- 每章至少有一个推动情节的关键事件

## 退出标准

每章/每卷完成时验证：
- 字数达标（≥大纲预期的 80%）
- 连贯性检查通过（角色、世界观、情节线无矛盾）
- 文笔质量通过同行评审
- 作品知识文件已更新（角色状态、情节推进）
- 创作进度表已更新

## 协作规则

- Writer 在收到 Worldbuilder 和 Character Designer 的产出后才能开始写作
- Plot Architect 的大纲修改需通知所有相关 Writer
- Continuity Manager 有权要求任何章节返工以保持连贯性
- Style Editor 的修改建议优先级高于进度要求
- Reader Analyst 的市场反馈用于调整后续章节方向，不改已发布内容

## 特殊规则

- 创作类任务允许更长的执行周期，不因短期无产出判定失败
- 灵感类任务（worldbuilding、character）允许多次迭代
- 紧急加更需求需 Chief 评估对质量的影响后决定
